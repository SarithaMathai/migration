# Jira — bulk-create the migration stories

Two ready-to-use formats (both regenerated by [`generate.py`](./generate.py)):

| File | Use it when |
|------|-------------|
| `all-stories.csv` | Create **every** epic + story across all 4 domains in one import. |
| `<domain>.csv` (`product.csv`, `bom.csv`, `measurement.csv`, `impression.csv`) | Import a single domain at a time. |
| `<domain>-stories.md` | Create issues **manually, one at a time** — copy a block into a new issue's description. |

> **Totals:** 4 epics + **149** stories (product 72, bom 42, measurement 24, impression 11).
> Each `<domain>.csv` has the domain's Epic as the first row, then one row per story.

---

## A. Bulk import a CSV (recommended)

Jira **Cloud**: *Filters → View all filters → Import issues from CSV* (or
*⚙ Settings → System → External System Import → CSV*).
Jira **Server/DC**: *⚙ → System → External System Import → CSV*.

1. Pick the project, upload `all-stories.csv` (or a single `<domain>.csv`).
2. On the **field-mapping** screen, map columns as below.
3. For the `Labels` columns (there are **three**, all named `Labels`), map **each** to the Jira
   *Labels* field — Jira merges them into the multi-value Labels field.
4. Run the import. Epics are created from the `Epic` rows; stories link to their epic via **Epic Link**.

### Column → Jira field mapping

| CSV column | Jira field | Notes |
|------------|-----------|-------|
| `Issue Type` | Issue Type | `Epic` or `Story` |
| `Story ID` | *(map to a label or a custom "Story ID" field, or leave unmapped)* | e.g. `SPARK-PROD-B01`; also prefixed into Summary for traceability |
| `Summary` | Summary | `SPARK-PROD-B01 · getProduct` |
| `Epic Name` | Epic Name | only set on Epic rows (required by Jira to name an epic) |
| `Epic Link` | Epic Link | set on Story rows = the epic's name; links story→epic |
| `Phase` | *(custom field or label)* | A–G migration phase |
| `Complexity` | *(custom field or label)* | Low / Medium / High / Very High |
| `Story point estimate` | Story point estimate | derived from complexity — **confirm in refinement** (see below) |
| `Labels` ×3 | Labels | `dgs-migration`, `<domain>`, `<type>` |
| `Depends On` | *(see §C)* | comma-separated Story IDs; not auto-linked |
| `Description` | Description | full story spec (Current Behaviour → Target → Files → Acceptance → Tests) |

> The CSV is UTF-8 with a BOM and standard quoting, so it also **opens cleanly in Excel** if you want
> to review/edit before importing.

### Story-point mapping (derived — confirm in refinement)

| Complexity | Points |
|------------|--------|
| Low | 2 |
| Medium | 3 |
| High | 5 |
| Very High | 8 |

These are AI-derived from complexity tiers only; the source stories carry **no** day estimates. Re-point
in backlog refinement. To change the mapping, edit `POINTS` in `generate.py` and re-run.

## B. Create issues manually (per-issue Markdown)

Open `<domain>-stories.md`, create the **Epic** first, then for each `## …` block:
create a Story, set its Summary to the block heading, and paste the block body into the Description.
Jira Cloud's editor accepts Markdown on paste; set Labels/points from the metadata line.

---

## C. Dependencies ("is blocked by")

The `Depends On` column lists prerequisite **Story IDs**. Jira CSV import can't create issue links to
issues that don't have keys yet, so links are **not** created automatically. After import, add links
in one of two ways:
- **Manually** — open a story, *Link → is blocked by*, and search by the Story ID (it's in the Summary).
- **Second CSV pass** — once issues exist, export their keys, build a small link-only CSV
  (`Issue key`, `is blocked by` = the blocker's key), and re-import.

## Regenerating

Re-run after any story edits, or after a **new domain** is analyzed (its `finalOutput/<domain>/`
folder appears):

```
python finalOutput/jira/generate.py
```

The script auto-discovers every `finalOutput/*/04-stories-index.yaml` — no code change needed for new
domains (claims, productDetails, watchlist, search, …). See
[`../scripts/skill-06-consumption-artifacts.md`](../scripts/skill-06-consumption-artifacts.md).
