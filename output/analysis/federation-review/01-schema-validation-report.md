## 2. Which schema is authoritative, and why

| Artifact | Role | Authoritative? |
|---|---|---|
| `output/analysis/{domain}/be-03-schema.graphql` | Federated **target** schema per domain | ✅ Yes — for the migration target |
| `code/schemas/*.txt` (SDL, `SPARK_*`/`VMM_*`/`IG_*`…) | **Current** internal gateway schema (source of translation) | ✅ Yes — for the *current* state; it is the input, not obsolete |
| `output/summary/*`, `output/confluence/*`, `output/jira/*` | Generated derivatives (regenerated by `generate_all.py`; hand edits overwritten) | ❌ Never edit |
| `output/analysis/program/fe-00…04, fe-09…11` | Generated FE derivatives (from `generate_frontend.py`) | ❌ Never edit |
| `output/analysis/program/fe-05, fe-08` + per-domain `be-01…be-05` md | Hand-authored analysis / story sources | ✅ Source of truth for stories |

- Rationale: `generate_jira.py` and `generate_frontend.py` both read `output/analysis/` as their single source (`UPDATED_SRC` and fallback point to the same path); each target schema header names its exact source SDL and resolver file, so lineage is traceable per domain.

## 3. Validation checks performed

- **Coverage** — every source operation is carried over: each schema's status legend reports `0 ✅ | N 🔜 | 0-3 ⏭` with all queries/mutations preserved by name (verified for all 8 files).
- **Key selection** — program rule (decided 2026-07-17): every entity federates on `id`. Where the SDL exposes only `humanId` (claims, packaging, watchlist, dieline), the entity wraps the record with a synthesized `id` (= humanId, the Measurement pattern) so stitching is uniform. Composite keys remain where identity genuinely is composite: `productId partnerId` (ResourcesCount), `sampleId` (SampleMeasurementSet), `id version` (TightFit, VersionableId).
- **Boundary tagging** — internal-vs-federated tagging is consistent with the monorepo decision (`plm-product` hosts product, bom, measurement, impression, packaging, productDetails, watchlist; claims is a separate `spark-claims` subgraph).
- **Cross-schema consistency** — comparing the 8 files against each other surfaced **5 contract mismatches** (type names, `@key` fields, entity-vs-value-type modelling) that would fail Hive/rover composition. These are the *required* findings of this review — see [03-proposed-schema-changes.md §1](./03-proposed-schema-changes.md) and [04-entity-analysis.md §3](./04-entity-analysis.md).
- **Standalone-compile caveat — ✅ fixed** — product's schema referenced cross-subgraph paged wrappers it never declared (`TeamPaged`, `TeamPagedV2`, `WorkspacesPagedV2`, `DiscussionElastic`, `MeasurementsPaged`). `BomPaged` needed no fix — it's declared by the co-located bom schema (fine in the merged `plm-product` schema). `MeasurementsPaged` was a naming mismatch, corrected to `MeasurementPaged` (R4). The other three (`TeamPaged`, `TeamPagedV2`, `WorkspacesPagedV2`, `DiscussionElastic`) are now declared as `@shareable` placeholder value types in `product/be-03-schema.graphql`, sized to today's field usage; `TeamPaged` is duplicated field-for-field in `claims/be-03-schema.graphql` (R5) and the two must stay in sync until the team subgraph (phase 2) becomes the single owner. Implemented in PRODUCT-BE-F-14.

## 4. Assumptions

1. The 5 non-phase-1 domains (attachment, discussion, sample, search, workspace) intentionally have no target schema yet; references to them stay gateway-stitched stubs until phase 2 (per `generate_jira.py` domain catalogue comment).
2. `code/schemas/core.txt` is the canonical definition for shared platform types (`VMM_BusinessPartner`, `VMM_Brand`, `SPARK_Categories`, paging types); where a per-domain SDL disagrees with a target schema, the target schema (resolver-verified) wins.
3. Dates in schema headers (2026-06-26/27) predate the story-file updates (2026-07-16); the story files carry later decisions (e.g. `Bom_Unified` removal is reflected in both — checked). No drift found between the bom/product schemas and their story files.
4. ACL is ignored in all DGS implementations (stakeholder decision recorded in every domain's §ACL) — this review does not re-open it.
