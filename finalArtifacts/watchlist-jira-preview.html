2 · How each row maps to a Jira issue

  • Story ID → appears in the summary as [WATCHLIST-BE-…] — this is how the push matches existing issues & links dependencies
  • Summary → issue summary (domain-prefixed)
  • Depends On → is blocked by issue links
  • Phase / T-shirt → labels phase-B, size-XS
  • Description → Acceptance Criteria + a Full story: GitHub back-link (nothing else)
  • Epic Link → attaches the story to its BE or FE epic
